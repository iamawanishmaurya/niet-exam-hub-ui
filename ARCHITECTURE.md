# Architecture

> Generated by /map on 2026-03-07

## Overview
NIET Exam Hub is a single-page application (SPA) built with React and TypeScript, powered by Vite. The application is designed to serve study materials, split between two primary modes: "Exam" and "PPT". It relies on Appwrite as its backend for handling user uploads and study material persistence, while using local JSON data for asset mappings. The UI is heavily componentized utilizing Shadcn UI (Radix UI) and Tailwind CSS.

## Data Flow
1. **Routing:** Users navigate via `react-router-dom` in `App.tsx`. The app handles sub-routes under `/exam/*` and `/ppt/*`.
2. **State/Mode Selection:** `ModeContext` determines whether the app is serving Exam papers or PPT materials.
3. **Data Fetching:** For static mappings, local `.json` files in `src/data/` are fetched and processed. For dynamic functionality (e.g. Uploading), requests are made via the Appwrite SDK to the `StudyMaterials` Appwrite cloud instance.

## Conventions
- **Naming:** PascalCase for components (`PaperCard.tsx`), camelCase for hooks and utilities (`useExamPapers.ts`, `appwrite.ts`).
- **Structure:** Feature-based routing separated by pages, shared generic components in `/ui/`, composite components in `/components/`.
- **Styling:** Tailwind CSS with a standardized set of `shadcn/ui` components.
- **Testing:** No apparent automated test suite (e.g. Jest/Vitest) is currently configured based on dependencies or directory structure.
